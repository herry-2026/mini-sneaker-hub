# 今日球鞋热搜 · AI 工作守则（AGENTS.md）

> 本文档告诉 AI 在写代码时要遵守哪些规矩。零基础学员不用全看懂，知道大概意思就行。

## 项目大概
- 前端：React + TypeScript + Vite + CSS
- 后端：Node.js + Express
- 平台：识货、得物、虎扑、Sneaker News、Hypebeast、Nice Kicks（共6个）

## AI 写代码时必须遵守的规矩

### 1. 代码风格
- 组件名用大写开头（如 `HotCard`）
- 函数名用小写开头（如 `fetchData`）
- 文件命名清晰，一看就知道是干什么的

### 2. 前端规矩
- 桌面版：卡片排成 3 列
- 手机版：卡片排成 1 列（自动适应）
- 排名第 1、2、3 名的文字用不同颜色（比如第1名红色，第2名橙色，第3名黄色）
- 如果某个平台数据加载失败，显示“加载失败，点我重试”按钮，不能整个页面都崩掉

### 3. 后端规矩
- 不能在前端直接去识货、得物等网站抓数据（会被拦住），必须走后端
- 抓数据时，请求头里要假装自己是浏览器（设置 User-Agent）
- 每个平台独立缓存，默认 10 分钟刷新一次
- 缓存代码写在 `server/utils/cache.js`

### 4. 安全与隐私
- 不要把密码、密钥等敏感信息写在代码里（用环境变量）
- 页脚必须加上：“本站为个人学习项目，数据来源于各平台公开内容，非商用”

### 5. 错误处理
- 一个平台抓不到数据，不影响其他平台展示
- 后端接口出错时，返回的 JSON 里要包含 `error: true` 和 `message: "出错了，请稍后重试"`
- 前端拿到错误后，只在该平台卡片上显示“加载失败”，其他卡片正常
