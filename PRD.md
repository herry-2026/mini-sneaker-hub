# 今日球鞋热搜 · PRD（优化版）

## 产品概述
一个多平台球鞋资讯/热搜聚合网站，首页以卡片网格展示国内识货网、得物、虎扑的热门内容，以及海外 Sneaker News、Hypebeast、Nice Kicks 的最新热门文章。用户打开链接即可浏览，无需登录。

## 目标用户
- 球鞋爱好者、潮流玩家，想快速了解各平台正在讨论的热门球鞋、发售信息
- 对 Vibe Coding / 全栈开发感兴趣的学习者

## 核心功能（MVP）

### 必须做
1. **首页卡片网格**
   - 6 张卡片，分为国内区（3 卡）和海外区（3 卡）
   - 每张卡片显示：平台名称、榜单名称（如“热门榜”“本周流行”）、Top 10～15 条
   - 每条包含：排名、标题、跳转链接（热度值可选，海外平台不展示热度值，可展示发布日期）

2. **数据接口（自建后端）**
   - `GET /api/hot/shihuo`
   - `GET /api/hot/dewu`
   - `GET /api/hot/hupu`
   - `GET /api/hot/sneakernews`
   - `GET /api/hot/hypebeast`
   - `GET /api/hot/nicekicks`
   - `GET /api/hot` 返回全部 6 个平台数据
   - 统一 JSON 格式（见技术设计）

3. **数据来源策略**
   - **国内平台**：优先寻找公开 JSON 接口；若无，则解析网页（需加 User-Agent、延迟 ≥2 秒，遵循 robots.txt）
   - **海外平台**：使用 RSS 订阅源（Sneaker News、Hypebeast、Nice Kicks 均提供 RSS）解析标题、链接及发布日期（如有）
   - 热度值（heat）：国内平台若有则展示；海外平台不展示该字段

4. **缓存策略（增强）**
   - 每个平台独立缓存，TTL 5～10 分钟（默认 600 秒）
   - 采用 **stale-while-revalidate** 策略：缓存过期后仍返回旧数据，同时后台异步更新，更新完成后前端下次刷新生效
   - 缓存 Key 设计：`cache:platform`，存储数据及上次成功更新时间戳
   - 页面展示「更新于 X 分钟前」基于上次成功更新时间
   - 提供开发环境下的手动清除缓存接口 `POST /api/cache/clear`（生产环境关闭）

5. **异常处理与重试机制**
   - 后端为每个平台请求设置独立超时（10 秒），超时后标记该平台失败
   - RSS/网页解析异常时捕获错误，返回空列表并记录日志
   - 单平台失败（网络错误、解析失败、无数据、反爬验证码）不影响其他卡片
   - 后端对失败平台自动重试 1 次（间隔 2 秒），仍失败则前端显示友好提示 + 「重试」按钮
   - 前端提供「全部重试」按钮，一键刷新所有失败卡片
   - 数据字段校验：每条记录必须包含 `title`（非空）、`link`（缺失则置为 `#`）、`rank`（缺失则用索引+1）；无效条目直接丢弃并记录警告
   - 后端使用 `Promise.allSettled` 并发抓取 6 个平台，避免串行等待

6. **健康检查与监控**
   - 后端增加 `GET /api/health` 端点，返回每个平台的最后成功抓取时间、下次计划抓取时间、最近错误摘要
   - 可配合 Uptime Robot 或 Railway 自带监控，当某平台超过 30 分钟无新数据时记录告警日志

7. **部署**
   - GitHub 托管代码
   - 公网 HTTPS 可访问（前端 Vercel，后端 Railway）

### 本期不做（明确边界）
- 用户登录 / 注册
- 搜索功能
- 历史热榜 / 收藏
- 分类 Tab（如“篮球鞋”“跑鞋”）
- 热度值爬取（海外平台不追求）
- WebSocket 实时推送

## 界面设计要求
- 简洁现代，信息密度类似「今日热榜」
- 桌面端：国内区 3 列 + 海外区 3 列（可分两行或在一行用 flex wrap）
- 移动端：1 列，卡片堆叠
- 海外卡片若 RSS 提供发布日期，则在每条标题后以小字显示（如“2小时前”），不展示排名列
- 页脚包含：学习项目、非商用、数据来源说明、更新频率、版权免责声明、隐私说明（不收集个人信息）、GDPR 提示（仅声明不使用追踪 Cookie）

## 合规声明（页脚及独立页面）
- **版权免责**：“本网站展示的内容版权归原始平台（识货、得物、虎扑、Sneaker News、Hypebeast、Nice Kicks）所有，本站仅为聚合展示，不存储任何数据文件。如认为内容侵权，请联系删除。”
- **数据来源说明**：明确列出各平台数据获取方式（RSS 或公开网页），并声明遵守 robots.txt 协议，模拟正常用户访问，设置抓取间隔。
- **隐私保护**：“本站无需登录，不收集任何个人信息。部署平台（Vercel/Railway）可能收集标准访问日志，详情请参阅各平台隐私政策。”
- **GDPR 合规提示**：在页脚或首次访问时提示“本站不使用追踪 Cookie”，符合欧盟基本要求。

## 技术栈（定稿）
- 前端：React + TypeScript + Vite + CSS
- 后端：Node.js + Express
- RSS 解析：`rss-parser`
- HTML 解析（必要时）：`cheerio`
- 缓存：内存 Map，配合 stale-while-revalidate
- 部署：前端 Vercel，后端 Railway

## 数据格式示例
```json
{
  "platform": "shihuo",
  "name": "识货热门榜",
  "lastUpdated": "2026-06-02T10:30:00Z",
  "items": [
    {
      "rank": 1,
      "title": "Nike Air Jordan 1 芝加哥复刻",
      "link": "https://...",
      "heat": 98234,
      "pubDate": null
    }
  ]
}
```
海外平台无 `heat` 字段，可增加 `pubDate`（ISO 字符串）。

## 附加说明
- 开发阶段可使用本地 mock 数据调试，上线前切换为真实抓取
- 建议为后端设置环境变量 `CRAWL_DELAY_MS=2000` 控制国内网页抓取间隔
- 若某平台长期不可用（如反爬升级），可在代码中临时禁用该平台（通过配置开关），不影响整体服务
